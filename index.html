Zieldefinition
Zweiradverleihsysteme sind Systeme, die es einem Nutzer oder Kunden erlauben, ein Zweirad zu leihen. Neben dieser grundlegenden Eigenschaft eines Zweiradverleihsystems ist die Definition sekundärer Ziele vonnöten, um eine zufriedenstellende Umsetzung dessen zu ermöglichen. Diese Ziele lassen sich zu verschiedenen Graden den wirtschaftlichen, ökologischen und sozialen Interessen der Stakeholder des Systems zuordnen.
Automatisierung
Mithilfe strenger Digitalisierung und Prozessautomatisierung kann der Personalaufwand des laufenden Systems minimiert werden. Automatisierte Systeme, die komplexe Aufgaben erfüllen, sind allgemein fehlerresistenter, skalierbarer sowie globaler anwendbar und damit wirtschaftlicher. Auch ist der Nutzeraufwand bei Verwendung eines entsprechend ausgelegtem automatisierten, digitalen Systems geringer.
Verwaltung
Zur Auswertung von Daten, Abrechnungen und Verwaltung der Zweiradflotte ist ein zentrales Verwaltungssystem vonnöten. Es ist mit einer örtlich stark verteilten sowie variierenden Last zu rechnen. Daher muss das Verwaltungssystem skalierbar und global unabhängig sein. Für Softwarelösungen wird meist auch ein Anspruch an Modularität gestellt. Dieser kann hier ducrh die Möglichkeit der Erweiterung auf andere Fahrzeuge, Bezahlmöglichkeiten abgebildet werden. Grundsätzlich sollte das System softwaretechnisch modern entwickelt werden, um eine langzeitliche Unterstützung und Wartung zu vereinfachen.
Wartung
Das System sollte möglichst wartungsarm sein. Dieses Ziel trifft sowohl auf Hard- wie auch Software zu. Zweiräder sollen eine maximale Dienstzeit haben, ein wartungsarmes Design wird also auf lange Zeit die laufenden Kosten senken. Eine Balance zwischen initialen Anschaffungskosten und laufenden Wartungskosten muss gefunden werden. Gleiches trifft auf Softwaremodule zu. Diese müssen zukunftsfähig entwickelt werden, um kritische Änderungen möglichst aufwandsarm zu ermöglichen. Sicherheitsupdates etc. sollten nicht durch die Steifheit der Software behindert werden. Eine ordentliche Programmierung und Dokumentation sind verpflichtend.
Wirtschaftlichkeit
Das System wird aus einem grundsätzlich wirtschaftlichen Interessen heraus geschaffen, daher ist eine Profitgenerierung unerlässlich. Nicht alle Funktionen müssen in ihrer Umsetzung auf wirtschaftlichkeit optimiert sein, summiert müssen sie dem Ziel jedoch gerecht werden.
Lageunabhägigkeit
Das System soll global eingesetzt werden. Daher müssen Komponenten lageunabhängig zur Ausführung ihrer Funktion fähig sein. Abhängikeit von eigens bereitgestellter Infrastruktur sollte im besten Falle nicht vorhanden, im schlechtesten minimal sein. Das Verwaltungssystem muss global erreichbar sein. Zahlungsmethoden müssen global zugänglich sein, also auch unter Umständen lokale Zahlungsmittel wie Girokarten akzeptieren. Das Zahlen mit international verwendeten Zahlungsmitteln wie PayPal, SEPA, BTC (o.Ä.) sollte unterstützt sein, um Spezialisierung auf lokale Zahlungsmittel möglichst zu vermeiden.
Datenschutz
Das System wird mit sensiblen sowie identifizierenden Daten von Nutzern arbeiten. Ortungsdaten, Identitätsdaten sowie Zahlungsdaten müssen vertrauclich behandelt werden. Zusätzlich müssen örtliche Datenschutzbestimmungen wie die DSGVO eingehalten werden.
